    ğŸ”§ Common Issues & Solutions

    Issue 1: "Module not found" errors

    Solution: Make sure you installed all dependencies:

    pip install -r requirements.txt

    Issue 2: "CUDA not available" warning

    Solution: This is OK! The system will use CPU. It's just slower. - If you have an NVIDIA GPU, install CUDA from: https://pytorch.org/

    Issue 3: Port 5000 already in use

    Solution: Stop other programs using port 5000, or change the port:

    # In api.py, change the last line:
app.run(debug=True, host='0.0.0.0', port=5001)  # Use 5001 instead

    Issue 4: Video upload fails

    Solution: Check that: - Video file is under 500MB - Format is supported: .mp4, .avi, .mov, .mkv, .webm - You have enough disk space

    Issue 5: Detection takes too long

    Solution: - This is normal for CPU processing (can take 1-2 minutes per video) - Use a GPU for 5-10x faster processing - Reduce video length/quality for faster testing

    ğŸ“ Understanding the Results

    When you analyze a video, you get:

    Key Metrics

    1. Verdict: AUTHENTIC or DEEPFAKE

    2. Based on AI analysis of facial features and artifacts

    3. Confidence Score: 0-100%

    4. How confident the AI is in its decision

    5. 80% = High confidence

    6. 50-80% = Medium confidence

    7. <50% = Low confidence (review manually)

    8. Authenticity Score: 0-100%

    9. Inverse of confidence (for authentic videos)

    10. Higher = More likely to be real

    11. Method: Which AI model was used

    12. ResNet50: Most reliable, CNN-based
    13. Enhanced: SOTA ensemble model
    14. CNN: Faster, basic classifier

    What to Do with Results

    • High Confidence (>80%): Trust the result
    • Medium Confidence (50-80%): Consider additional verification
    • Low Confidence (<50%): Video quality may be too low, or it's edge case
